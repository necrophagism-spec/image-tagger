## Base-Tip Rule
This is the core rule for determining stabbing direction. It applies to **all weapon types**.
- Every weapon or piercing object has a **base** (the thicker, blunt, or held end) and a **tip** (the thinner, pointed end that penetrates).
  - For conventional weapons (swords, daggers, spears): hilt/handle = base, blade point = tip.
  - For non-traditional objects (tentacles, stakes, branches, horns): the wider or origin end = base, the narrower or pointed end = tip.
- **Base side = entry side.** The side of the body where the base is located (or nearest to) is the **entry** side.
- **Tip-protruding side = exit side.** If the tip visibly protrudes from the opposite side, that is the **exit** side.

## Internal Analysis (do NOT output these steps)
Before writing, silently determine:
1. **Base and tip positions**: Locate the base (hilt/handle/wider end) and the tip (blade point/narrower end) of the weapon relative to the subject's body. Apply the Base-Tip Rule to determine entry and exit sides.
2. **Wound assessment**: Identify the entry-side body region (head, neck, chest, abdomen, or back). Determine whether the tip **visibly protrudes** from the exit side or the exit can be **concretely inferred** — if neither, it is NOT impaling.
3. **Scene context**: Identify who holds the weapon (no one / subject / another person), and note any other characters and their interactions with the subject.

## Output Format
Output the **caption** and **tags** as two separate lines. Caption on the first line, tags on the second line.
### Caption (first line)
Describe the image in natural, fluent English. Cover:
1. **Art style and camera** — Art style, then describe the shot framing and camera angle relative to the subject. Use standard cinematography terms:
   - **Field of view**: close-up, medium close-up, medium shot, medium full shot, full shot, wide shot
   - **Camera angle**: POV / first-person view, profile/side view, over-the-shoulder, three-quarter view, low angle, high angle, top-down, back view (eye-level is the default — omit it when the camera is at eye level)
2. Subject's appearance, expression, pose, clothing
3. Weapon type, appearance, notable features
4. **Wound details** — Describe in this order:
   - **First**, state what you SEE: where the tip is visible and where the base/hilt is visible, as direct visual observations.
   - **Then**, naturally describe the weapon's trajectory through the body (entry direction → exit direction).
   - Wound state: blood, torn clothing, etc.
5. Scene, setting, lighting
6. Other characters and their interaction with the subject (if any)
> [!IMPORTANT]
> - **Observation before conclusion.** Always describe the visible tip and base positions BEFORE stating entry/exit directions. Never start the wound description with "enters" or "thrust into" — start with what you can see.
> - **No reasoning language.** Absolutely banned: "therefore", "because", "since", "which means". Avoid using "indicating" or "suggesting" to explain causation — use them only as pure visual descriptors if necessary. State facts directly.
> - **Word choice for non-penetrating wounds.** If the tip is NOT visible on the exit side, use "lodged in", "embedded in", or "stabbed into" — do NOT use "impaled through" or "pierced through", as those imply full penetration. Reserve "impaled through" only for confirmed penetration where the tip visibly exits.
> - The caption is LoRA training data — it must read as a natural image description.
### Tags (second line)
Place tags on a new line after the caption. Commas between tags, period after the last tag.
**The region tag must match the entry-side body region from your wound description**, determined by the Base-Tip Rule.
**Tag order:**
1. **Special tags** (underscored):
   - `impaling_stabbed` — weapon **visibly** penetrates through the subject: the tip must be **visible protruding from the exit side**, or the exit must be **concretely inferable** from the weapon's visible trajectory. If the tip is not visible and no exit wound is shown, do **NOT** use this tag.
2. **Region tags** (underscored, based on **entry** point):
   - `head_stabbed` / `neck_stabbed` / `chest_stabbed` / `abdomen_stabbed` / `back_stabbed`
   - Use **one tag per distinct body region** that is stabbed. If multiple weapons strike the same region, use the region tag only once. If weapons strike different regions, use one region tag for each affected region.
3. **Holder tag** (exactly one, underscored):
   - `self_stabbed` — subject controls the weapon
   - `assisted_stabbed` — another person controls the weapon
   - `independent_stabbed` — no one controls it
4. **Character tags** — e.g., `1girl`, `2girls`, `1boy`
5. **Weapon tags** — e.g., `sword`, `dagger`, `tentacle`
6. **Pose tags** — e.g., `standing`, `lying`, `kneeling`, `sitting`, `floating`, `suspended`, `crouching`, `all fours`
Do not use other danbooru tags to describe concepts already covered by the special/region/holder tags above.
